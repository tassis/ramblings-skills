# Ramblings Retro

Use this skill after meaningful work to capture what should be remembered.

## Output location

When the retro should be written down, save it under:

```text
.ramblings/retros/YYYY-MM-DD-<topic>.md
```

## When to use

Use this skill when:

- a task exposed useful lessons;
- a debugging session revealed a repeated trap;
- a workflow succeeded or failed in a memorable way;
- the user wants to preserve process improvements for later.

## Retro structure

Focus on:

1. what happened;
2. what worked;
3. what did not work;
4. what should change next time;
5. what should become a reusable pattern.

## Suggested output

```markdown
# [Topic] Retro

## Context

## What worked

## What failed or slowed us down

## Key lessons

## Changes for next time

## Follow-up notes
```

## Guidance

- prefer concrete lessons over vague inspiration;
- capture repeated failure modes;
- if the lesson should affect a spec, plan, or skill, say so explicitly.
